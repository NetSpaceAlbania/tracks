## What is Shell and Terminal?
	- A shell interprets user commands and communicates with the OS to execute those commands
	- Provides a command-line interface for interacting with the computer's operating system
	- The terminal provides a text-based interface for interacting with the shell
	- Allows the user to enter commands and view the results of those commands

## Basic Shell Commands:
- cd: Change directory
- ls: List files and directories
- mkdir: Create a new directory
- touch: Create a new file
- rm: Remove a file or directory
- mv: Move a file or directory
- cp: Copy a file or directory
- cat: Concatenate files and display them
- grep: Search for a pattern in a file

## Filesystem Structure:
- The filesystem is organized in a hierarchical structure
- The root directory is at the top
- Directories and files are arranged in a tree-like structure
- Each file and directory is identified by a path that starts at the root directory and ends with the file or directory name

## Permissions:
- A set of rules that dictate who can access a file or directory and what actions they can perform on it
- Categorized into three groups: owner, group, and other
- Each group has specific permissions, such as read, write, and execute
- The owner of a file or directory can modify the permissions to grant or revoke access to other users

## Volumes and Persistence:
- A volume is a physical or logical storage device that contains files and directories
- Volumes can be configured for persistence, allowing data to be stored on the device even when the computer is turned off
- Persistence is important for applications that need to store data for long periods of time, such as databases or file servers
- Volumes can also be used to store backups of important data to protect against data loss

## Installing an OS without External Media:
- It is possible to install an operating system without external media by creating a bootable USB drive or DVD
- Some operating systems also have built-in tools for creating bootable drives
- It is also possible to install an operating system from within an existing operating system by using virtualization software

## Binaries and Configs:
- Binaries are executable files that contain machine code that the computer can execute
- Configs are configuration files that specify settings for an application or system
- Binaries and configs can be modified to customize the behavior of an application or system
- Modifying binaries and configs can introduce security risks or cause the software to malfunction

## Ricing:
- Ricing refers to the process of customizing the appearance and behavior of the user interface on a computer
- Typically involves modifying settings for the shell, terminal, and other applications
- Can include changes to fonts, colors, backgrounds, and other visual elements
- Can also involve creating custom scripts or plugins to extend the functionality of the software.
